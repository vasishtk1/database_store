Distributed KV Store

Architecture & Tech Stack Reference

Tech Stack

Language & Runtime
Go 1.21
Goroutines for concurrency, built-in net package, same lang as etcd & CockroachDB
sync.RWMutex
Many concurrent readers OR one writer — protects the in-memory map
goroutines
Election timer, heartbeat loop, apply loop, per-client TCP handler
Networking
net/rpc
Node-to-node RPC (RequestVote, AppendEntries) — standard library, zero deps
encoding/gob
Auto-serialises RPC structs over TCP (used by net/rpc under the hood)
net.Listen / Dial
Raw TCP for client-facing server on :8080; separate port per Raft node
Storage
bbolt (BoltDB)
Single-file embedded DB. Used for both the Week 1 WAL and Raft's durable state
encoding/json
Serialises WAL entries & Raft log to BoltDB. Human-readable, easy to debug
map[string]string
The actual in-memory KV store — a plain Go map behind an RWMutex
CLI & Tooling
cobra
Subcommand CLI (kvctl get / put / delete). Same lib used by kubectl & gh
flag (stdlib)
--addr and --wal flags on the server binary
math/rand + time
Randomised election timeouts (150–300 ms) prevent split-vote elections

File Map

File Week Purpose
internal/store/store.go Week 1 In-memory map[string]string with RWMutex. The state machine.
internal/wal/wal.go Week 1 BoltDB write-ahead log. Appends every mutation to disk; replays on restart.
internal/server/server.go Week 1 TCP server. Parses text commands, dispatches to WAL + Store. Will route writes through Raft in Week 3.
cmd/kvctl/main.go Week 1 CLI client. Cobra subcommands: get, put, delete. Opens TCP, sends command, prints response.
main.go Week 1 Entry point. Opens WAL → replays into Store → starts TCP server.
internal/raft/log.go Week 2 LogEntry (command in the Raft log) and ApplyMsg (committed entry ready to run).
internal/raft/rpc.go Week 2 RequestVote and AppendEntries request/reply structs — the two RPC wire types.
internal/raft/persister.go Week 2 Saves currentTerm, votedFor, and log to BoltDB atomically before every state change.
internal/raft/node.go Week 2 Central Node struct. New(), Stop(), Submit(). Helper methods used by every other raft file.
internal/raft/election.go Week 2 Election timer, startElection(), RequestVote handler. How the cluster recovers from a crashed leader.
internal/raft/replication.go Week 2 Heartbeat loop, AppendEntries handler, commit-index advancement. How writes propagate to followers.
internal/raft/apply.go Week 2 Apply loop. Polls commitIndex, sends ApplyMsg on channel → bridges Raft to the KV store.
internal/raft/transport.go Week 2 RPC server (net/rpc over TCP) and callRPC() with timeout + connection caching.
